How I Build Software

  • Ruby on Rails

    Ruby on Rails is my framework of choice for building web applications. It enables immense productivity by giving developers a toolbox full of well-integrated components like Action Pack, Active Model, and Active Record. Rails minimizes boilerplate code by leveraging Ruby’s elegant, human-friendly syntax and powerful metaprogramming capabilities.

    My approach to Rails development is to stay as close to “stock” Rails as possible, so I can take full advantage of the synergy between its components. I’m not a fan of gems like GraphQL Ruby that significantly change a project’s structure. Instead, I follow the “fat models, skinny controllers” guideline and prioritize building an expressive domain model over introducing service classes. I believe that a well-crafted domain model is one of the greatest assets in a Rails application.

    I practice Test-Driven Development (TDD), as described in 99 Bottles of OOP . TDD helps me design clear interfaces, codify requirements, and focus on solving the problem at hand. I strive for the simplest possible solution, avoiding unnecessary abstractions and speculative features in order to keep the codebase readable, maintainable, and adaptable.

  • PostgreSQL

    I wholeheartedly agree with the statement from the PostgreSQL homepage: “The World’s Most Advanced Open Source Relational Database.” Relational databases remain one of the best ways to ensure consistent access to data in concurrent environments. PostgreSQL’s internals make it especially well-suited for applications where data is read more often than it is written, which is a common pattern in web development.

    PostgreSQL’s approach to concurrency control is uniquely elegant. By providing Serializable Snapshot Isolation, it delivers both correctness and strong performance under high concurrency. Beyond its rock-solid core, PostgreSQL offers a rich set of features, indexing strategies, and operations. Extensions like pgvector or PostGIS make it possible to handle specialized use cases without introducing additional databases to deploy, monitor, and pay for.

    Recently, I’ve been diving deeper into PostgreSQL’s capabilities. My current favorite resource (besides the excellent official documentation) is High Performance PostgreSQL for Rails .

  • Hotwire

    Hotwire gives Rails developers a major productivity boost. With Turbo, it allows backend developers to progressively enhance their applications with features typical of Single Page Applications—without writing a single line of JavaScript. By controlling client-side interactions through server-rendered templates, Hotwire provides a straightforward way to implement modern, dynamic features.

    When simple DOM updates aren’t enough, Stimulus can be used to add custom JavaScript functionality in a clean, Rails-like structure. Hotwire Native extends this approach even further by enabling deployment of Rails applications to Android and iOS. The ability to reach the web and all major mobile platforms simultaneously is a huge advantage for teams that need to serve a broad customer base.

    Hotwire Native for Rails Developers and Master Hotwire are excellent books that help build a solid mental model of the technologies.
